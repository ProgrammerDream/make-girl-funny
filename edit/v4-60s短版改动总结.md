# v4 — 60s 抖音短版优化总结

> 在踩坑出片（v3）基础上做的二次迭代。目标：**总片长压到 60s 内、首帧炸、字幕和元素不被抖音 UI 遮挡**。

---

## 改动一览

| 文件 | 改动类型 | 一句话总结 |
|---|---|---|
| `script.json` | 大砍 | 每段从 8~16s 压到 5~7s，总时长 ≈ 60s |
| `extras/hook.html` | 重做 | 首帧大爆点 + VS + 底部 hashtag，预留抖音 UI 空间 |
| `extras/awards.html` | 提速 | 动画延迟全部前移，从 8s 缩到 5s 也能播完 |
| `build.js` | 改版式 | 字幕条从底部搬到顶部钢印下方，避开抖音 UI |
| `record.js` | 加功能 | `--only` 参数支持只重录指定段，不用全跑一遍 |
| `README.md` / `package.json` / `package-lock.json` | 重命名 | `girl-fun` → `make-girl-funny` |

---

## 1. script.json — 总片长砍半

### 为什么砍

抖音黄金时长是 60s 内，v3 拼起来超过 110s，前 3s 不够炸就划走。

### 怎么砍

每段只留**最炸的一个动作**，前后等待时间统一压短：

| 段 | 标签 | 旧时长 | 新时长 | 砍了什么 |
|---|---|---|---|---|
| 1 | Gemini 极简 | 8s | 5s | 删掉 hover 第 3、4 次和 modal 二次点击 |
| 2 | 豆包 油腻 | 8s | 5s | 鼠标轨迹从 6 段折线砍到 3 段 |
| 3 | Qwen 爱心分裂 | 9s | 5s | 心点击次数 9 → 6 |
| 4 | Grok 直球 | 13s | 6s | 删 confession / kiss / fortune，只保留 crazyHearts |
| 5 | GPT 克制 | 13s | 6s | 删 yes / mini buttons / secret，留 chase 追逐 |
| 6 | DeepSeek 戏精 | 13s | 6s | chase rounds 12 → 4 |
| 7 | MiniMax 卷王 | 16s | 6s | 6 张 card → 2 张（最炸的两张） |
| 8 | Claude 老婆手册 | 15s | 6s | 删 fortune / generateLove / chase / 6 张 food，留 2 张 food |
| 9 | Kimi 七大模块 | 16s | 6s | 删 generateCompliment / gameStart / holdBtn |
| 10 | GLM 烟花封神 | 15s | 7s | chase rounds 14 → 3，烟花停留 7s → 4.5s |

**合计**：134s → 58s。

`hook` 5s → 3s，`awards` 8s → 5s。

## 2. hook.html — 首帧爆点重做

### v3 的问题

- 标题用 `fadeUp 0.8s ease 0.2s`，**前 0.2s 是空白**。抖音前 0.2s 没东西就划走了。
- prompt 1.2s 才出，groups 2.2s 才出，2s 内信息密度不够。
- 用 `flex center`，元素挤在屏幕中间，**底部抖音 UI（作者名/原声/进度条）会糊一片**。

### v4 的解法

**首帧炸**：

```css
.boom { animation: boomIn .35s cubic-bezier(.2,1.5,.4,1) both, wobble 2s ... }
@keyframes boomIn {
  0%   { opacity:0; transform: scale(.4) rotate(-6deg) }
  60%  { opacity:1; transform: scale(1.1) rotate(2deg) }
  100% { opacity:1; transform: scale(1) rotate(0) }
}
```

`both` 关键字让动画在 0 帧就应用 `from` 状态，**第一帧就有内容**（虽然是 scale .4，但已经能看见）。

**节奏全部前移**：

| 元素 | v3 出现时间 | v4 出现时间 |
|---|---|---|
| 标题 | 0.2s | 0s（boomIn 即时启动） |
| prompt | 1.2s | 0.4s |
| groups | 2.2s | 1s（左右飞入） |
| VS 标记 | — | 1.5s（新增） |
| hashtag | — | 2s（新增「⚠️ 看到第 7 个我笑出声」钩子） |

**避开抖音 UI**：

```css
body { padding: 80px 40px 320px }
```

底部留 320px，作者名 / 原声条 / 进度条 / 互动按钮都在这块预留区里，主要内容不会被遮。

## 3. awards.html — 动画提速

颁奖榜旧时长 8s，v4 录屏砍到 5s。如果不动动画延迟，**会在最后一张卡片刚弹出时就被截断**。

把所有 `animation-delay` 前移：

| 元素 | v3 delay | v4 delay |
|---|---|---|
| h1 标题 | 0.1s | 0s |
| card 1 | 0.5s | 0.25s |
| card 2 | 0.8s | 0.45s |
| card 3 | 1.1s | 0.65s |
| card 4 | 1.4s | 0.85s |
| outro | 1.8s | 1.2s |

5s 录满，所有元素都能看到完整动画。

## 4. build.js — 字幕条搬家

### v3 的问题

字幕条在画面底部：

```js
// v3
drawbox=x=0:y=${VH - 220}:w=${VW}:h=220   // 底部 220px 黑条
drawtext=...:y=${VH - 160}                  // 文字 y
```

抖音 UI 也在底部：作者名、原声、进度条、点赞收藏。**两层叠在一起一片糊**。

### v4 的解法

字幕条搬到**顶部钢印下方**（钢印在 y=40~170）：

```js
const CAP_BAR_Y = 200 * SCALE;          // 字幕条顶边在 y=200
const CAP_BAR_H = 180 * SCALE;          // 高度 180
const CAP_TEXT_Y = CAP_BAR_Y + 56 * SCALE;
```

字号从 68 → 64（顶部条比底部条窄一点，字也跟着小一档）。

底部完全空出来给抖音 UI 和页面里的烟花/按钮。

## 5. record.js — 加 `--only` 单段重录

### 为什么

调一段动作经常要反复改 `script.json` 看效果。全部重录一次 4~5 分钟，**改一个段重录全部太亏**。

### 怎么用

```powershell
# 只重录 hook
node record.js --only hook

# 只重录 hook + awards + 第 4 段
node record.js --only hook,awards,seg-04

# 只重录第 4 和第 7 段
node record.js --only seg-04,seg-07
```

不传 `--only` 就和原来一样全部重录。

实现是一个 `should(name)` 闭包过滤循环：

```js
const should = (name) => !only || only.has(name);
if (should('hook')) { await recordOne({ name: 'hook', ... }) }
for (const seg of SCRIPT.segments) {
  const name = `seg-${String(seg.id).padStart(2, '0')}`;
  if (!should(name)) continue;
  await recordOne({ name, ... });
}
```

录完直接跑 `node build.js`，build 阶段会自动用 `raw/` 里现有的所有 mp4。

## 出片流程（v4）

跟 v3 一样，三步：

```powershell
node record.js          # 全部重录，约 3~4 分钟（v3 是 4~5 分钟，因为每段短了）
node build.js           # 拼接 + 钢印 + 字幕，约 1 分钟
start out\final.mp4
```

**调段技巧**：先 `node record.js --only hook` 单独验证 hook，OK 再跑全量。
